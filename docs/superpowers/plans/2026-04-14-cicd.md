# CI/CD Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a GitHub Actions workflow that runs lint on every push to `main` and deploys to the production server only when lint passes.

**Architecture:** Two jobs — `test` (lint gate) and `deploy` (SSH to server). The `deploy` job has `needs: test` so it only runs when lint passes. The lint script is fixed to target `app src` only so it doesn't flag minified third-party files. The deploy script mirrors what's currently run manually.

**Tech Stack:** GitHub Actions, `appleboy/ssh-action@v1.2.0`, `pnpm/action-setup@v4`, Node 22, pnpm 10

## Task 1: Fix the lint script

**Files:**
- Modify: `package.json`

The current `"lint": "eslint"` runs ESLint on everything, including minified files in `node_modules` and generated Prisma client code. This produces 790 errors that are all false positives and would make the CI test job always fail.

- [ ] **Step 1: Read the current lint script**

Open `package.json` and find the `"scripts"` section. It currently reads:

```json
"lint": "eslint",
```

- [ ] **Step 2: Fix the lint script**

Change it to:

```json
"lint": "eslint app src",
```

This scopes ESLint to the `app/` and `src/` directories — the only directories that contain project source code.

- [ ] **Step 3: Verify lint passes on source files**

Run:
```bash
pnpm lint
```

Expected: exit code 0, no errors (warnings are OK). If errors appear, they are real issues in `app/` or `src/` that must be fixed before proceeding.

- [ ] **Step 4: Commit**

```bash
git add package.json
git commit -m "fix: scope lint script to app src directories"
```

## Task 2: Create the GitHub Actions workflow

**Files:**
- Create: `.github/workflows/deploy.yml`

- [ ] **Step 1: Create the workflow file**

Create `.github/workflows/deploy.yml` with this exact content:

```yaml
name: Test & Deploy

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Lint
        run: pnpm lint

      # Uncomment when tests are added (sub-projects B/C/D):
      # - name: Test
      #   run: pnpm test

  deploy:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - name: Deploy via SSH
        uses: appleboy/ssh-action@v1.2.0
        with:
          host: bergholm.net
          port: 2222
          username: toni
          key: ${{ secrets.SSH_DEPLOY_KEY }}
          script: |
            set -e
            cd /home/toni/apps/positivenews
            git pull origin main
            pnpm install --frozen-lockfile
            pnpm build
            pm2 restart positivenews
```

- [ ] **Step 2: Commit**

```bash
git add .github/workflows/deploy.yml
git commit -m "feat: add CI/CD workflow — lint gate + SSH deploy on push to main"
```

## Task 3: Add the SSH deploy key secret (manual)

**This task is a manual step in the GitHub web UI — no code to write.**

The `deploy` job authenticates to the server using an SSH private key stored as a GitHub Actions secret named `SSH_DEPLOY_KEY`. The same key is already set up in the `tonibergholm/tom-cruises.com` repo.

- [ ] **Step 1: Copy the existing private key**

The key already exists in the `tonibergholm/tom-cruises.com` repo secrets. Go to:

```
https://github.com/tonibergholm/tom-cruises.com/settings/secrets/actions
```

The secret is named `SSH_DEPLOY_KEY`. GitHub does not let you read the value back, so you need to look up the original private key file.

The key pair was likely generated when setting up the tom-cruises.com deploy. Find it locally (e.g. `~/.ssh/bergholm_deploy` or similar) or generate a new one:

```bash
ssh-keygen -t ed25519 -C "github-actions-deploy" -f ~/.ssh/bergholm_deploy -N ""
cat ~/.ssh/bergholm_deploy.pub
```

If generating a new key, add the public key to the server:

```bash
ssh -p 2222 toni@bergholm.net "echo 'PASTE_PUBLIC_KEY_HERE' >> ~/.ssh/authorized_keys"
```

- [ ] **Step 2: Add the secret to this repo**

Go to:
```
https://github.com/tonibergholm/positivenews/settings/secrets/actions/new
```

- Name: `SSH_DEPLOY_KEY`
- Value: the contents of the private key file (the full text including `-----BEGIN OPENSSH PRIVATE KEY-----` header and footer)

Click **Add secret**.

- [ ] **Step 3: Verify the secret exists**

Go to:
```
https://github.com/tonibergholm/positivenews/settings/secrets/actions
```

Confirm `SSH_DEPLOY_KEY` appears in the list.
